Durée: 2 min

A qui s'adresse ce lab ?

Ce lab s'adresse aux équipes souhaitant moderniser des applications Java EE existantes, exécutées sur des serveurs traditionnels comme :

  • IBM WebSphere Application Server Version 6.1, ou ultérieure
  • WebLogic Version 6.x+
  • Apache Tomcat Version 6.0 (et ultérieure)
  • Red Hat Version JBoss 4.x+.

Objectif : Ce lab démontre comment les outils IBM automatisent leur transformation vers des architectures modernes.

Pourquoi moderniser ?

Les applications Java EE Legacy ont été développées il y a plusieurs années, souvent basés sur des technologies anciennes (Java EE, serveurs d'applications monolithiques, frameworks obsolètes). Ces applications présentent souvent des limites :

  • Maintenance complexe & coûteuse
  • Performance non optimisée pour le cloud
  • Risque de sécurité (liées aux versions obsolètes)
  • Faible scalabilité
  • Intégration difficile avec les technologies récentes

Quels outils IBM pour les moderniser ?

Dans ce tutoriel, nous utiliserons deux outils puissants d'IBM :

🔍 IBM Application Modernization Accelerator (AMA) - analyse le code source d'applications Java existantes pour :

  • identifier les dépendances et incompatibilités
  • estimer l'effort de migration
  • faciliter la refactorisation
  • générer un plan de modernisation

🤖 IBM Bob - an AI-augmented SDLC (Software Development Lifecycle) partner - un agent IA qui gère tout le cycle de vie du logiciel - il ne s'agit pas d'un simple Assistant IA mais d'un partenaire de travail.

Il s'appuie sur une combinaison de modèles d'IA; les modèles Granite d'IBM, mais aussi d'autres modèles pour utiliser le plus adapté à chaque étape de votre projet.

Couplé d'un add-on Java Premium Package, Bob automatise la modernisation :

  • refactorisation du code Java
  • exécution des builds et tests
  • déploiement auto sur Liberty
  • accompagne sur les meilleures pratiques
  • génération de documentation

💡 AMA analyse, Bob exécute.

Ce que vous allez apprendre dans ce lab:

  • Analyser d'anciennes applications JavaEE avec IBM AMA
  • Interpréter les rapports de modernisation
  • Utiliser IBM Bob pour automatiser la refactorisation du code
  • Valider l'application sur Liberty

Actions automatiques :
